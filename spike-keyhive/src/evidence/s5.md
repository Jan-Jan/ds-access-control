## Flow E1 — Authorise incoming session (Member principal)

There is no published Keyhive transport crate at this pin — Beelay is
not yet published. The spike implements its own minimal session model
in `s5_p2p_policy.rs`:

```text
pub struct PolicyManager<R: MemberKeyResolver> {
    resolver: Arc<R>,
    adapter: IdAdapter,
    sessions: Mutex<HashMap<SessionId, SessionRecord>>,
}

impl<R: MemberKeyResolver> PolicyManager<R> {
    pub fn authorise_session(
        &self,
        peer_vk: &VerifyingKey,
        doc: DocumentId,
    ) -> Result<SessionId, PolicyError> {
        // 1. Reverse-lookup peer_vk -> MemberId via adapter (cached entries
        //    only; cold lookups need a trie reverse-index in Phase 3).
        let member_id = self.adapter.member_id_for(peer_vk)
            .ok_or(PolicyError::UnknownPeer)?;
        // 2. Is this member authorised against `doc` via the trie?
        if !self.resolver.is_authorised(&member_id, &doc) {
            return Err(PolicyError::Unauthorised);
        }
        // 3. Allocate session record; return id.
        let id = SessionId::new();
        self.sessions.lock().unwrap().insert(id, SessionRecord {
            peer_vk: *peer_vk, member_id: Some(member_id),
            doc, state: SessionState::Open,
        });
        Ok(id)
    }
}
```

## Flow F1 — Terminate sessions on member revocation

The `MembershipListener::on_revocation` callback fires after a
revocation is processed. The spike's listener impl extracts the
revoked `Identifier` from `Signed<Revocation>` and:
1. Reverse-looks up the affected doc(s).
2. Calls `policy_manager.recheck_open_sessions()` which walks the
   in-memory session map and flags any session whose `(peer_vk, doc)`
   pair is no longer authorised.

```text
impl<F, S, T> MembershipListener<F, S, T> for PolicyListener {
    fn on_revocation<'a>(&'a self, data: &'a Arc<Signed<Revocation<F, S, T, Self>>>)
        -> F::Future<'a, ()>
    {
        F::pure(async move {
            // Extract revoked member from the signed op
            let revoked_id = data.payload().revoke.identifier();
            // Trigger session recheck via the shared policy manager
            self.policy_mgr.recheck_open_sessions();
        })
    }
}
```

## Flow E2 / F2 — Org principal

Same shape, with the wrapper resolving `Principal::Org` through the
gate-4 mechanism: `authorise_session_for_org` checks whether
`member_id` is currently a member of the org (via
`org_group.transitive_members().contains_key(&member_id.into())`),
and if so, whether the org has access to the doc.

## Reverse-lookup gap

`adapter.member_id_for(&peer_vk)` only resolves entries that have
been cached by a prior call. A cold authorise request from a peer
the spike has never seen has no entry — the spike falls back to
returning `PolicyError::UnknownPeer`.

In production this gap requires a formal
`MemberKeyResolver::find_member_by_device(VerifyingKey) -> Option<MemberId>`
method (or trie reverse-index) that the trie populates. The same
gap was recorded for p2panda's gate-5 spike; it's a foundation-layer
gap, not a Keyhive-specific issue. Phase 3 should formalise this as
an addition to the `MemberKeyResolver` trait in `spike-common`.

## Push-vs-pull termination

Termination is push-based via `MembershipListener::on_revocation`,
not pull-based as in p2panda's spike (which used periodic
`recheck_open_sessions` because no push hook was available at the
sync layer). Keyhive's listener-driven model is materially better for
revocation latency: a revocation propagated through `receive_static_event`
immediately fires the listener and triggers the recheck. p2panda has
latency proportional to the recheck cadence.

## Comparison vs p2panda

p2panda's gate 5 recorded the same four `Soft / TraitImpl Small`
rows with two architectural concerns:
- Pre-open window between iroh TLS handshake and `p2panda-sync`
  policy check.
- Pull-based termination via periodic `recheck_open_sessions`.

Keyhive's gate 5 records the same Soft rows but without those two
concerns. Net: slightly cleaner mechanism, similar burden.
